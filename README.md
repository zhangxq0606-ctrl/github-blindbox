# GitHub 每日盲盒

每天 18:00，AI 自动从 GitHub Trending 筛选**真正值得关注的项目**，按「创意好玩 / 工具 / AI 应用」三类整理好，发到你 QQ 邮箱。

**服务器我已经搭好了，数据源我包了。** 你不需要任何服务器，Fork 我的仓库，加2个 Secret，就能收邮件。

## 数据来源：多语言覆盖，去重聚合

绝大多数 Trending 工具只看单一语言榜，视野狭窄。我们每天用 **5 个语言维度并行抓取**：

| 维度 | 用途 |
|------|------|
| `(全语言)` | 覆盖今日整体趋势 |
| `python` | 数据科学 / AI 生态 |
| `javascript` | 前端 / Node.js 生态 |
| `typescript` | 现代全栈生态 |
| `go` | 云原生 / 基础设施生态 |

5 路数据去重合并后，得到 **50-80 个候选项目**，再交由 AI 做第二轮筛选。这样既不会漏掉冷门语言的好项目，也不会因为单一维度的噪声错过真正值得关注的东西。

### 它和别的 Trending 产品有什么不同？

其他 GitHub Trending 相关产品（RSS 订阅、邮件简报、排行榜网站）做的事本质上就是一句话：**把热门 repo 列出来给你看**，排序方式无非是按星数或今日新增。你是那个被动接收信息的人，筛选全靠自己。

**我们做了三层过滤，不是简单的"挑几个好的"：**

**① 多源去重聚合 → 建立候选池**  
通过 5 个语言维度（`python`、`javascript`、`typescript`、`go` 及全语言）分别抓取 Trending，拿到 50-80 个不重复项目。这一步解决了单语言 Trending 视野狭窄的问题。

**② 历史去重 → 避免信息疲劳**  
自动排除前几天已经推送过的项目，保证每天都是新鲜内容，不会重复看到同一个项目。

**③ 人格化 AI 筛选 → 按"人"过滤，不是按"指标"**  
这是最核心的一步。AI 不是简单按星数排序，而是按照一个**具体的读者画像**来筛选：

> 独立开发者，vibecoding 实践者，有产品思维和运维思维，追求实用、有创意的产品。不需要底层技术、算法、框架等纯开发内容，喜欢「能直接用的」和「有意思的」。

选完后 AI 强制按三个类别输出——**🎨 创意/好玩 → 🛠 工具 → 🤖 AI 应用**，每个类别必须有 2-3 个项目，不能跳过任何类别。这意味着 AI 必须主动去找它可能本来不会注意到的项目类型（比如创意类），而不是只捡星数最高的那几个。

> ### 🧑‍🎨 这里的筛选画像，你可以自己改
> 
> **这就是整个项目最核心的定制点。** Fork 后打开 `config/preferences.json`，改 `readerProfile` 字段（阅读者画像）和 `hardFilters` 数组（数据层硬过滤规则）——比如你是前端开发者、游戏爱好者、独立开发者还是 AI 研究员，AI 就会完全按你的口味来筛。
> 
> - `readerProfile`：写你自己的画像，AI 据此筛选项目
> - `hardFilters`：在送 AI 之前直接删掉明显不相关的项目（正则匹配 description 或 fullName），不浪费 AI token
> 
> 想扩大筛选范围就写松一点，想精准命中就写细一点。**这是你的项目，怎么筛你说了算。**
